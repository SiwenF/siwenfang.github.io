OKX Flexible Loan Journey Revamp

A data-driven redesign that unified single and multiple collateral loan flows, resulting in significant improvements across key metrics.

Role: Product Designer • Platform: Mobile App • Duration: 3 Months
↓

01 — Discovery

Identifying the Problem

Through data analysis and user research, I identified five critical problems in the existing loan flow that were directly impacting conversion rates and user satisfaction.

  1. Multiple collateral loan had a significantly lower conversion rate: over 1/5 of single collateral loan's conversion rate
  2. Multiple collateral loan had significantly longer time to convert: almost 3.5x longer time
  3. 7.53% of all users (over 1,300 users) applied for both single collateral loan and multiple collateral loan
  4. >80% customer complaints were about basic loan subscriptions and repayment related issues

02 — Original Design

Existing Subscription Flow

The original design separated single collateral and multiple collateral loan flows, creating a fragmented user experience with lower conversion rates.

Most users have 2 Single collateral loan orders at a time. It indicates that multiple collateral loan is in demand

03 — Exploration

Design Exploration

Three design directions were explored, each testing a different hypothesis on how to unify the loan experience.

Direction 1 — Unified Entry Point

Merge single and multi-collateral into one flow. The core user action is identical; the only variable is the number of collateral assets selected.

Hypothesis

One consistent mental model eliminates the need to choose a loan “type” upfront, reducing decision fatigue.

Direction 2 — Goal-First Chunking

Flip the flow: let users state what they want to borrow first, then figure out collateral. Splits the cognitive task into two focused steps.

Hypothesis

Leading with the borrowing target creates forward momentum and reduces upfront friction.

Direction 3 — Progressive Disclosure

Start with the user’s top currency pre-selected. Multi-asset selection only appears when they tap “+ Add more collateral,” keeping the primary screen clean.

Hypothesis

Exposing complexity only on demand keeps the default experience simple while supporting power users.

04 — Final Design

Complete Flow

The revamped design unifies the loan experience — users can now seamlessly borrow with single or multiple collateral in one streamlined flow.

05 — Comparison

Before & After

A side-by-side comparison of the original fragmented experience versus the unified redesign.

Before

Separate flows for single and multi-collateral loans. Complex navigation between loan types. Users confused by redundant interfaces. Low discoverability of multi-collateral option.

After

Unified subscription flow for all loan types. Streamlined collateral selection with pre-filled defaults. Clear LTV indicators and risk information. One-tap access to borrow with any collateral combination.

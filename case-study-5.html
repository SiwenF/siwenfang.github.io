03

The conversion problem

How do we visually convey a “conversion” happening instantly during a transfer?

I had some UI exploration, trying to solve the issue via clear UI. E.g. in the preview page, on top of the page, highlighting the token switch.

⚡

Show the conversion at the first step,
not the last.

Unifying input amount and surfacing conversion there helps users understand when they start — not at preview when it’s too late to reconsider.

Unified title: “Transfer USDG” Same UI for In & Out Less dev effort

04

Week 2 — Edge cases

Changes are always expected in crypto.

Problem: conversion triggers at different stages for Transfer In vs Transfer Out — two different experiences in one surface.

Transfer In
  1. User confirms
  2. Conversion starts
  3. Passkey auth
  4. Transfer complete

Conversion before auth = risky

Transfer Out
  1. User confirms
  2. Passkey auth
  3. Conversion starts
  4. Transfer complete

Auth before conversion = safer

The loading page dilemma

Add the error page only

Add a loading friction

Split preview into 2 steps (convert > transfer)
